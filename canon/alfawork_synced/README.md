# Iskra Canon Documentation — Sync Pack

Этот архив содержит **две** ветки документации:

- `CURRENT_SOT_FLAT19/` — **текущий канон (SoT)** в плоском формате `00–18` (совместим с ChatGPT Projects).
- `PRODUCTION_DOCS/` — **рабочая документация по доработке стека**. Содержит стратегию, правила валидации и инструкции по интеграции.
- `LEGACY_DOCS/` — **наследованные документы** (старые ревизии/черновики). Они сохранены для истории и для продолжения доработки и дополнения CURRENT_SOT_FLAT19 , но **не считаются источником истины**, если расходятся с CURRENT.

## Как читать (быстро)

1) Начни с `CURRENT_SOT_FLAT19/00_INDEX_AND_ROUTING.md`.
2) Для самопроверки целостности — `CURRENT_SOT_FLAT19/17_AUDIT_INTEGRITY_CHECK.md`.
3) Для технической валидации (CI/CD) используй `canon/tools/validate_canon.py`.
3) Если смотришь старые тексты — относись к ним как к “архиву”: сравнивай утверждения с CURRENT.

## Политика разрешения конфликтов

Если `LEGACY_DOCS/*` противоречит `CURRENT_SOT_FLAT19/*`, истинным считается CURRENT.
Причина: CURRENT — это **единственная** согласованная плоская поставка под Projects (flat19).

- Документация, где фигурировали “7 граней/голосов”, помечена как **legacy** (текущий канон фиксирует реестр **9 голосов** в File 03).
- Интегрити‑отчёт `17_AUDIT_INTEGRITY_CHECK.md` приведён к фактической плоской поставке (flat19): корректный список файлов и sha256‑манифест SoT.
