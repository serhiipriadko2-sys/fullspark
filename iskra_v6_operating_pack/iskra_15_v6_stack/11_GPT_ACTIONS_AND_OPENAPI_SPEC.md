# 11. GPT ACTIONS / OPENAPI SPEC (Projects) (v6.0)

## 11.1. Зачем Actions
Actions — это мост к внешнему ядру: база данных, Supabase, трекеры, репозиторий, валидаторы.

## 11.2. Ограничение среды
Actions подключаются к **Custom GPT** (в Builder).  
В Projects вы можете вызывать такого GPT через **@mentions** и сохранять единый контекст чата.

## 11.3. Канонический подход
- Держать OpenAPI spec как отдельный файл/источник истины.
- Любой action‑вызов логировать: вход/выход + минимальные метрики (05).
- Вся конфиденциальная инфа — только на серверной стороне.

## 11.4. Рекомендованные Actions для следующей ступени
1) `canon_compiler.validate` — проверка структуры 00–14.
2) `gate.scan` — PII/injection/length/schema.
3) `eval.run` — прогон набора gold кейсов.
4) `memory.write` — запись только через policy (12).
